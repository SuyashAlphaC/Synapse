Walrus in every tick.
Not a bolt-on demo.

The Walrus track asks for agents that remember, share context, and produce verifiable artifacts. Synapse runs that pipeline on every autonomous tick — rebalance or noop — on testnet Fargate vaults today.

01 · memory

MemWal every tick

recall before decide, remember after. Counters, EMA state, and decision history survive restarts. Same semantic query in the dashboard recall panel.

02 · artifacts

Walrus audit trail

Markdown rationale uploaded every tick. On-chain ArtifactRef stores blob id + SHA-256. Any aggregator can verify — hash mismatch means tampering.

03 · coordination

Multi-agent by default

Shared MemWal namespace with on-chain CrossAgentReadEvent. Sui Stack Messaging for rebalance signals — Seal payloads on Walrus, correlator on-chain.

04 · safety

Move policy + attestation

Spend cap, allowlist, expiry, one-tx revoke. Optional Nautilus path: enclave signs the decision; Move verifies before DeepBook executes.

<how> · recall → reason → act → remember

Six steps.
One Walrus-native loop.

Every vault tick composes MemWal recall, strategy evaluation, Move-gated execution, Walrus artifact publish, optional peer coordination, and MemWal remember — not as optional hooks, but as the default runtime path.

  1. 01
    mint

    Mint with zkLogin

    Sign in with Google. Spawn a vault with spend cap, contract allowlist, expiry, and optional Walrus execution consent baked into Move at mint time.

  2. 02
    recall

    Recall via MemWal

    Before every decision the runtime recalls semantic memory from Walrus — counters, past rationale, and cross-agent peer facts from a shared namespace.

  3. 03
    execute

    Act on DeepBookV3

    One PTB: optional attestation verify → policy gate → DeepBook swap → performance + royalty. The Move VM rejects anything off-policy — atomically.

  4. 04
    publish

    Publish to Walrus

    Upload tick rationale as a markdown blob. On-chain ArtifactRef stores blob id + SHA-256 so anyone can verify the audit trail independently.

  5. 05
    coordinate

    Signal peers

    On rebalance, emit a Seal-encrypted signal via Sui Stack Messaging. On-chain correlator links digest to channel — peers consume it as memory on the next tick.

  6. 06
    remember

    Remember & revoke

    Persist the outcome to MemWal for the next tick. One owner revocation PTB cascades session authority, MemWal delegate, and Walrus eviction — instantly.

<faq>

Frequently asked.

What happens if the AI agent goes rogue?

The owner clicks Revoke vault. A single PTB sets AgentIdentity.revoked = true, invalidates the MemWal delegate, and queues Walrus blobs for eviction. Total time from "rogue" to "dead": one transaction. No multisig, no vendor approval.

How are the policy gates enforced?

In the Move VM, not in our backend. Every spend call checks four invariants atomically: not revoked, sender = session key, before expiry, target on allowlist — plus the per-epoch spend cap. A compromised agent runtime cannot bypass these.

Who runs the Synapse Vault?

You do. Synapse Core is open source and the strategy runner is a Node process you host. We don't custody your funds, we don't see your data. The compliance trail is verifiable from any Sui fullnode.

Why 1% AUM?

Industry standard. Identical economics to Yearn Vaults, BlackRock ETFs, traditional asset management funds. We didn't invent a fee structure — we ported a proven one to autonomous agents.

Is this production-ready?

Testnet today — hosted Fargate ticks, live vaults, Walrus artifacts on every tick. External security audit (OpenZeppelin or OtterSec) before mainnet. Read THREAT_MODEL.md and the Walrus Track executive summary.

How do I integrate?

Mint a Vault at /mint. For your own strategy logic, fork the LangGraph adapter at @synapse-core/adapter-langgraph and run the Node CLI. Full setup in docs/runtime-setup.md.
